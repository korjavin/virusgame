# vs-ai2.49: Quiescence at contact leaves

## Overview
Fix exchange mispricing + contact passivity by adding bounded QUIESCENCE at
1v1 minimax leaves in `backend/search/search.go`. Today `minimax` at `depth==0`
calls the static eval immediately, so it prices a position with a *pending*
retaliation capture as if that capture will never happen. vs-ai2.47 proved the
resolved exchange already scores correctly (~-388 for a 1-for-2); the error
lives strictly at intermediate contact leaves. The structural cure is to, at a
leaf in a contact position, extend *capture-only* moves (stand-pat + bounded
alpha-beta) until the position is quiet, then evaluate. The eval then always
sees quasi-resolved positions and the mispricing vanishes.

This should flip the constructed bad-trade scenario to DECLINED at production
budgets and raise the bot's contact-attack counts (fixing the owner-observed
passivity).

## Context (from discovery)
- Target file: `backend/search/search.go` (leaf is `minimax` depth==0, lines
  ~207-210; `maxN` depth==0 lines ~305-308).
- A **capture** = a `Move` action whose `Target` cell is an enemy `Normal`
  (the only legal move targets are Empty or enemy-Normal; Base/Fortified are
  never legal targets, so captures are exactly Moves onto enemy Normal cells).
  Enumerate via `game.NewPosition(state).ForEachSearchAction`, keep only
  `action.Kind == game.Move` whose target is an enemy Normal; apply with
  `pos.ApplySearch(action)`.
- Turn structure: up to 3 moves per turn; `state.CurrentPlayer()` already
  drives side alternation (a capture decrements `movesLeft`; the engine
  advances the turn). Quiescence just recurses on capture children — the turn
  mechanics decide who moves next. Maximizing node ⇔ `state.CurrentPlayer() == s.root`.
- Eval: `evaluateWithWorkspace(state, s.root, &s.eval)` returns a score from the
  root's perspective (higher = better for root). Scale: `mateScore = 1e9`;
  a single capture ≈ ±388; space term ≈ ±222/cell.
- Node accounting: every `minimax`/`maxN` node does `s.nodes++`; `s.running()`
  stops at `nodeLimit`. Quiescence nodes MUST be counted in `s.nodes` so
  node-budget modes stay honest/deterministic.
- Determinism contract: a search depth either completes fully or is discarded
  (`complete=false` propagates up). Quiescence must honor this — return
  `(int, bool)` like `minimax`, with `(0,false)` when `!s.running()`.
- DO NOT touch `backend/search/evaluate.go` (parked vs-ai2.44) or
  `backend/search/incumbent/*` (BYTE-FROZEN).
- maxN (multiplayer, >2 players) is left AS-IS: all gates are 1v1; multiplayer
  needs only no-regression, which is free if maxN is unchanged.
- Coordination: dev/vs-ai2.39-tiebreak-jitter touches ROOT selection
  (`atDepth`); this work is in the LEAF area (`minimax` depth==0). Keep the diff
  out of root selection so the merge stays clean.

### Task 1: Add bounded capture-only quiescence to 1v1 minimax leaves
- [x] In `backend/search/search.go`, add a `quiescePlyCap` const (start `6`
      capture plies) and an optional env-tunable delta margin
      `quiesceDelta` (default `0` = delta pruning DISABLED; cap is the primary
      cost bound). Read the env override once in `newSearcher` if present
      (e.g. `VS_QUIESCE_CAP`, `VS_QUIESCE_DELTA`) so sweeps are possible but the
      shipped default is a fixed constant — mirror existing tunable patterns.
- [x] Add method `func (s *searcher) quiesce(state game.State, alpha, beta, ply, qdepth int) (int, bool)`:
      - `if !s.running() { return 0, false }`; `s.nodes++`.
      - `if state.GameOver() { return terminalScore(state, s.root, ply), true }`.
      - stand-pat: `s.evaluations++; standPat := evaluateWithWorkspace(state, s.root, &s.eval)`.
      - `maximizing := state.CurrentPlayer() == s.root`.
      - Fail-soft stand-pat cutoff: maximizing → if `standPat >= beta` return
        `standPat,true`; if `standPat > alpha` set `alpha = standPat`.
        minimizing → symmetric on `beta`.
      - `if qdepth >= s.quiesceCap { return standPat, true }`.
      - Enumerate capture-only children in stable order (see Task 1 note):
        `game.Move` actions whose target is an enemy `Normal`. If none, return
        `standPat, true` (position is quiet).
      - For each capture child: optional delta prune (maximizing:
        `if s.quiesceDelta > 0 && standPat + s.quiesceDelta <= alpha { continue }`);
        `score, ok := s.quiesce(child, alpha, beta, ply+1, qdepth+1)`;
        `if !ok { return 0, false }`; standard fail-soft alpha-beta update; break on `alpha >= beta`.
      - Return best (fail-soft), `true`. Do NOT read or write `s.table` (keep
        quiescence TT-free for determinism/simplicity).
- [x] Add a small helper to enumerate capture children deterministically:
      iterate `game.NewPosition(state).ForEachSearchAction`, keep only
      `action.Kind == game.Move` where `c,_ := state.At(action.Target)` has
      `c.Kind == game.Normal && c.Owner != state.CurrentPlayer()`, append
      `child{action, pos.ApplySearch(action).State(), ...}`. Order is board-order
      (stable) — no TT move needed. Honor `s.running()` inside the loop (return
      incomplete on stop) to preserve the determinism contract.
- [x] Wire the leaf: in `minimax`, replace the `if depth == 0 { s.evaluations++; return evaluateWithWorkspace(...), true }`
      block with `if depth == 0 { return s.quiesce(state, alpha, beta, ply, 0) }`.
      (quiesce does its own `s.nodes++`/`s.evaluations++`; do NOT double-count —
      note the current depth==0 path does `s.nodes++` at function entry then
      `s.evaluations++`; keep the entry `s.nodes++` for the leaf minimax node and
      let quiesce add its own nodes for the extension. Verify counts in Task 2.)
- [x] Leave `maxN` and the `len(children)==0` static-eval fallback unchanged.

## Technical Details
- `quiesce` signature mirrors `minimax`'s `(int, bool)` complete-contract.
- Fail-soft alpha-beta, stand-pat cutoff, cap on capture plies (default 6),
  optional delta pruning (default off). No TT reads/writes in quiescence.
- Determinism: capture enumeration is board-order stable; node budget honored
  inside the capture loop; identical inputs ⇒ identical Nodes/Evals/Score.
- Node budgets: quiescence `s.nodes++` per node keeps deterministic node-budget
  modes honest — goldens re-pinned to the true counts.
