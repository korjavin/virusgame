# vs-ai2.56 — NNUE-lite data generator + trainer (stages 1-2)

## Overview

Bead **vs-ai2.56** stages 1-2 of Plan B2: build the offline pipeline that will
later feed a learned evaluation network. This is DATA + TRAINING only.
Stage 3 (pure-Go int8 inference wired into `evaluateWithWorkspace`) is a
separate later bead and is explicitly OUT of scope here.

Two deliverables plus a feature extractor and docs:

1. A **feature extractor** that reproduces, arena-side, the cheap per-player
   quantities `backend/search/evaluate.go` computes.
2. A **data generator** (`backend/arena/cmd/nnuegen`) that samples positions
   from self-play / owner-corpus / ladder openings, labels each with a deep
   search score + eventual game outcome, and writes deterministic JSONL shards.
3. A **trainer** (`tools/nnue-train`) — a self-contained, dependency-free Go
   program that trains a small MLP on the shards and exports int8-quantized
   weights as Go source, with a loader stub for Stage 3 to adopt.
4. **Docs** with the full-run commands, JSONL schema, weights format, honest
   feature-coverage notes, and observed smoke loss-curve numbers.

## Context (from discovery)

- **Frozen search must not change.** `backend/search/evaluate.go` and
  `backend/search/search.go` are OFF LIMITS. The analysis machinery
  (`playerMetrics`, `analyze`, `analyzeWithConnectivity`, `spaceRace`,
  `connectedCellsInto`, `articulationPointsInto`) is all UNEXPORTED, so the
  feature extractor MUST mirror those cheap computations arena-side using only
  the public `game.State` API (`At`, `Rows`, `Cols`, `Active`, `CurrentPlayer`,
  `MovesLeft`, `NeutralUsed`, `GameOver`, `Winner`, `LegalActions`, `Apply`,
  `Snapshot`). This mirroring is the architect's explicit directive; do NOT add
  an exported analysis function to package `search`.
- **Exported search API for labels:** `search.ChooseNodeBudget(state, limit) (Result, bool)`
  and `search.ChooseDepth(ctx, state, depth)`; `Result.Score` is the deep score.
- **Agents** live in `backend/arena/agents.go`: `TelemetryNodeBudget(nodes, frozen)`,
  `TelemetryTournament(depth)`, `Greedy`, `BaseAttacker`, `MobilityAttacker`,
  `Legacy(seed)`, `Random(seed)`, and `Instrument(agent)` to wrap a plain Agent.
- **Data sources already exported arena-side:**
  - `arena.RandomLegalOpening(rows, cols, seed) (game.Snapshot, error)` — ladder openings.
  - `arena.LoadOwnerCorpus(path) ([]OwnerCorpusEntry, error)`,
    `arena.DecodeReplay`, `arena.ReplayPositions(replay) (map[ReplayPoint]game.State, error)` — replays.
  - `backend/arena/testdata/owner-corpus.json` + `testdata/*.json` replay fixtures.
  - `arena.StateFingerprint(state) (string, error)` — stable dedupe key (already exported).
- **Template:** `backend/arena/cmd/corpusgen/main.go` shows the deterministic
  xorshift64 + flag + JSON-writer style to mirror.
- **Module:** `backend/go.mod` = module `virusgame`, go 1.24; deps limited to
  uuid, gorilla/websocket, modernc sqlite. No gonum. Trainer stays stdlib-only.

## Testing Strategy

- **Unit tests** (stdlib `testing`, table-driven where natural):
  - feature extractor: a drift-guard test asserting hand-computed counts on
    fixed small boards.
  - nnuegen: a tiny in-process generation asserting JSONL round-trips and is
    byte-identical across two same-seed runs (determinism).
  - trainer: a few-epoch train asserting loss strictly decreases and the int8
    export round-trips through the loader within quantization tolerance.
- **No E2E** — this is offline tooling with no UI.

## Technical Details

- **Feature vector**: fixed field order per player; the trainer consumes the
  mover's perspective (current player's features minus opponents', or the full
  4×K concatenation — decide in Task 2 and document in the schema).
- **Label**: `deepScore = Result.Score` from `ChooseNodeBudget` at a high
  offline budget; `outcome` = eventual winner + placement from the source game
  (0/unknown for corpus/ladder positions with no completed game — record a
  sentinel and let the trainer mask the auxiliary term).
- **Determinism**: all randomness from seeded xorshift64 keyed by `-seed` +
  worker index; shard bytes reproducible across runs.
- **Quantization**: symmetric per-matrix int8 (`q = round(w / scale)`,
  `scale = max|w| / 127`); loader reconstructs `w ≈ q * scale`.
