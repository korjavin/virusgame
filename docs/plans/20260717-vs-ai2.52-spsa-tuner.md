# vs-ai2.52 — SPSA tuner for eval constants against the gate ladder

## Overview
Tune the ~19 hand-set evaluation constants in `backend/search/evaluate.go` by
SPSA (Simultaneous Perturbation Stochastic Approximation) black-box
optimization against the CORRECT objective — the deterministic gate ladder
(stranglers + hybrids + incumbent h2h + small-board floors) with SPRT-style
early stopping. The prior ML tuning line died measuring vs the incumbent (the
wrong opponent, which shares the strangulation weakness); this measures win-rate
vs stranglers, where eval quality actually shows.

Deliverables:
1. Eval constants exposed as a package-level parameter vector, injectable ONLY
   by the tuner in-process; production path stays byte-equivalent (defaults ==
   current constants, proven by the existing oracle/golden tests).
2. A reusable, non-test ladder harness in package `arena` (SPRT early stop,
   parallel game workers) so a cmd can drive the exact gate objective.
3. `spsatune` cmd: reproducible SPSA loop with antithetic perturbations, hard
   floors as reject constraints, CutSeeker held out for validation only.
4. A bounded smoke run with committed trace + best vector, honestly reported.

### Task 3: SPSA tuner cmd (backend/arena/cmd/spsatune)
- [ ] Create `backend/arena/cmd/spsatune/main.go`. Flags: `-iters` (25),
      `-openings` (8), `-nodes` (1000), `-seed` (1), `-workers`
      (GOMAXPROCS), `-out` (results JSON path), plus small-board floor knobs
      `-floor-openings` (default e.g. 6) with sane documented constants.
- [ ] Model `theta` as `EvalParams` in float form; helper to round to int
      `EvalParams` for injection. SPSA gains: `c_k = c/(k+1)^gamma`,
      `a_k = a/(k+1+A)^alpha` with documented defaults (e.g. a=... , c=... ,
      A=... , alpha=0.602, gamma=0.101 — the Spall standard); pick c large
      enough that `c_k` rounds to a nonzero integer perturbation for the
      integer-scaled params (scale per-param by its default magnitude so a ±
      step is meaningful for both weight=1 and BaseThreat=650 params).
- [ ] Each iteration k: seed an rng from `(seed, k)`; draw Bernoulli ±1 `delta`
      per param; `thetaPlus=theta+c_k*delta`, `thetaMinus=theta-c_k*delta`
      (antithetic); `fPlus=fitness(thetaPlus)`, `fMinus=fitness(thetaMinus)`
      evaluated SEQUENTIALLY (activeEvalParams is a process-global: set global,
      run all that eval's games via the parallel worker pool, collect, then the
      other). `g_i=(fPlus-fMinus)/(2*c_k*delta_i)`; `theta += a_k*g`. Track best
      `theta`/fitness seen (evaluate baseline `theta0` fitness at k=0 for the
      before/after comparison).
- [ ] `fitness(params) float64`: `search.SetEvalParams(round(params))`. FLOORS
      FIRST (any breach ⇒ return a large negative sentinel, candidate rejected,
      and record which floor breached): small-board Legacy ≥85% and Greedy ≥75%
      (candidate `TelemetryNodeBudget(nodes,false)` vs `Instrument(Legacy(1))`
      and `Instrument(Greedy)` on 8x8 via `PlaySequentialOpenings` with
      `-floor-openings`), and incumbent h2h ≥50% (candidate vs
      `TelemetryNodeBudget(nodes,true)` on 12x12). Then ladder score = weighted
      sum of candidate win% over 12x12 rungs {Greedy, Legacy, BaseAttacker,
      MobilityAttacker, MobilityBaseAttacker, incumbent-h2h} each via
      `PlaySequentialOpenings(...,50,8,...)`; document the weights (heavier on
      the stranglers per the memory that that is where eval quality shows).
- [ ] HOLDOUT: measure CutSeeker win% via `PlaySequentialOpenings` and record it
      per iteration but EXCLUDE it from the fitness sum (validation only).
- [ ] Always `defer search.SetEvalParams(search.DefaultEvalParams())` so the
      process global is restored.
- [ ] Trace: write a JSON file to `-out` — an array (or JSONL) of per-iteration
      records `{k, thetaPlus, thetaMinus, fPlus, fMinus, floorsOK, floorBreached,
      holdoutCutSeekerWinPct, bestFitness, bestTheta}` plus a final
      `{baselineFitness, bestFitness, bestTheta, defaultTheta}` summary. Print a
      compact per-iteration line to stdout.
- [ ] `go build ./...` and `go vet ./...` clean.
- [ ] Reproducibility self-check: add a tiny deterministic check (a
      `main_test.go` in the cmd, or a `-selfcheck` mode) that runs 2 SPSA
      iterations at `openings=2, nodes=200` twice with the same seed and asserts

## Technical Details
- `EvalParams` is a flat int struct; `activeEvalParams` a package global. Only
  `SetEvalParams` mutates it, and only the tuner calls it. Byte-equivalence is
  structural: production never injects, defaults equal the old literals, so the
  oracle digest is unchanged.
- SPSA maximizes fitness (win-rate weighted ladder). Antithetic pair per
  iteration = 2 fitness evals. Floors are reject constraints, not soft
  penalties. CutSeeker is never in the fitness path.
- The parallel worker pool lives at the games-within-one-fitness-eval level
  only; the two antithetic evals are serial because the eval params are a
  process global. `ponytail: process-global eval params ⇒ serial antithetic
  evals; goroutine-local params would allow full parallelism if the overnight
  run needs it.`
- Determinism sources: node-budget agents (no wall clock), fixed opening seeds
  and permutation seed, seeded per-iteration perturbations. No wall-clock
  anywhere in the objective.
