# vs-ai2.38: ungated own-fragility penalty — fix width-1 tendril openings

## Overview

The bot opens every game with a width-1 diagonal tendril (10,10/9,9/8,8 …) racing
to the center, and loses production games when the human cuts the filament at one
joint and rides the severed distal chain into the base (prod anchors
b543fe02 / bbfc5e0c, plus 4 more today). Root cause is in
`backend/search/evaluate.go`:

- The space-race term (`normalized(space, area, spaceRaceWeight=32)`) pays
  ~222/first-reached-cell for deepest-first expansion — the diagonal is
  Voronoi-optimal, so it is rewarded.
- Own structural fragility is **FREE pre-contact**: `threatenedLoss` (the term
  that prices losing a cut subtree) is GATED on `threatenedByConnected` (line
  ~337) — it only fires once an enemy cell is adjacent. The predatory cut bonus
  (lines ~213-218) is opponent-only. Nothing prices *own* fragility before
  contact, so a width-1 chain (every cell an articulation point) is built for
  free.

**Fix (red-team primary recommendation):** subtract each player's own MAX
single-cell `cutLoss` UNGATED — not the sum (over-penalizes), not a width prior
(too blunt) — scaled to compete with the space term. The data already exists:
`analyzeWithConnectivity` fills `workspace.cutLoss[index]` with the subtree
forfeited if that articulation point is cut (base excluded, 0 at non-articulation
cells). Max over that slice = the largest single-cut forfeit = exactly the
fragility of a filament. Sweep the coefficient; freeze the winner as a constant.
